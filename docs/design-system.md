# 白泽明理 Baize Tech 官网设计系统

版本：v1.0  
日期：2026-07-03  
适用范围：白泽明理官网、服务页、明理笔记、方法论页、案例页、联系页，以及后续品牌型 HTML 原型

## 1. 设计目标

白泽明理的官网设计系统要同时表达三件事：

1. 可信：面向企业服务，不能像轻浮的 AI 工具模板。
2. 可解释：信息结构清楚，让用户快速理解服务和方法。
3. 有记忆点：用“白泽”神兽 IP 建立区别于普通 AI 公司的识别度。

参考 Sakana AI 的极简首屏、强品牌符号、可交互图形场、克制导航和留白，同时参考 Sakana Fugu 产品页的红色主强调、sticky header、左侧 rail label、1px grid cells、monospace 小标签和低阴影信息结构。白泽明理应呈现“东方神话符号 + 可解释路径图形 + 严谨 AI 方法 + 企业落地服务”的组合。

## 3. 视觉方向

### 3.1 设计原则

- 少即是多：页面要留白，不堆积装饰；首页第一屏只做品牌建立，不塞服务卡片。
- 信息先行：服务、方法、转化路径必须清楚。
- 符号克制：白泽 IP 要成为记忆点，但不能变成卡通站。
- 技术不冰冷：用数理、证明、路径、图谱表达严谨感。
- 企业可信：避免花哨渐变、大面积蓝紫、过度科技风。

### 3.2 主视觉概念

推荐主视觉概念：

> 白泽在复杂信息迷雾中梳理出可解释路径。

视觉元素可以包括：

- 白泽线稿或印章式符号。
- 细线网络。
- 解释路径。
- 结构化节点。
- 低对比度数理符号。
- Logo 橙色关键点。
- Logo 蓝色与天蓝色可信辅助。

不建议：

- 赛博蓝紫渐变。
- 大量发光球体。
- 机器人形象。
- 随机抽象粒子背景。
- 过度拟人化卡通吉祥物。

| Token | 用途 | 色值 |
| --- | --- | --- |
| Success | 稳定、完成 | `#0F62B5` |

### 4.4 色彩使用规则

- 页面大底使用 White，长页交替段落使用 `#F9FAFB`。
- 内容区、表单、文章容器保持白底，并用 1px hairline 分隔。
- Fugu Red 用于 CTA、当前状态、链接和少量关键强调。
- Logo Orange 只用于品牌圆点和极少量视觉记忆点，不承担系统级交互色。
- Logo Blue 用于可信、安全、流程、方法论相关内容。
- Logo Sky 用于路径、辅助强调和局部视觉呼吸。
- 不允许使用大面积蓝紫渐变作为品牌主视觉。

## 5. 字体系统

### 5.1 字体栈

中文与英文混排：

```css
--bz-font-display: "Poppins", "Heiti SC", "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", sans-serif;
--bz-font-sans: "Poppins", "Heiti SC", "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", sans-serif;
--bz-font-mono: "JetBrains Mono", "SF Mono", "Menlo", "Cascadia Code", ui-monospace, monospace;
```

当前版本不依赖外部字体请求，`Poppins` 和 `JetBrains Mono` 作为可自托管字体名预留；中文实际落到 Heiti / PingFang / Hiragino Sans GB。首页中文品牌名使用较轻字重，避免变成海报标题；正文与导航使用系统无衬线和 monospace 小标签，保持服务信息清晰。

### 5.2 字阶

| Token | 用途 | 尺寸 | 行高 |
| --- | --- | --- | --- |
| Display | 产品页大标题 | 86px | 0.98 |
| H1 | 页面标题 | 56px | 1.04 |
| H2 | 大板块标题 | 40px | 1.18 |
| H3 | Cell 标题 | 18px | 1.5 |
| Body | 正文 | 16px | 1.9 |
| Small | 辅助说明 | 14px | 1.65 |
| Caption | 标签、元信息 | 12px | 1.4 |

移动端首页品牌名保持 36px 左右，服务页 H1 可降到 40px，正文仍需保证 15px 以上。移动端不要把长中文判断句做成巨型标题，优先保证呼吸感。

### 5.3 排版规则

- 中文正文行高不低于 1.7。
- 英文短语可作为辅助，但不要让英文抢主叙事。
- 标题不使用负字距。
- 大标题最多 2 行，避免堆叠口号。
- 正文行宽控制在 34-42 个中文字符。

## 6. 间距与布局

### 6.1 栅格

- 页面最大宽度：1280px。
- 内容阅读宽度：760px。
- 宽屏左右留白不小于 48px。
- 移动端左右留白 20px。

### 6.2 间距 Token

| Token | 值 | 用途 |
| --- | --- | --- |
| Space 4 | 4px | 极小间距 |
| Space 8 | 8px | 标签内部 |
| Space 12 | 12px | 紧凑元素 |
| Space 16 | 16px | 表单、卡片内部 |
| Space 24 | 24px | 卡片组 |
| Space 32 | 32px | 模块内间距 |
| Space 48 | 48px | 小板块间距 |
| Space 72 | 72px | 页面板块间距 |
| Space 104 | 104px | 首页大板块间距 |

### 6.3 布局规则

- 首页首屏采用接近 Sakana 首页的锁屏式品牌首屏：顶部轻导航、中心 Logo、动态图形场、短主张、底部 utility。
- 服务页与方法论页采用 Fugu 产品页结构：sticky header、产品式 hero、左侧 rail label、右侧正文。
- 不使用卡片套卡片。
- 服务信息优先使用 1px cell grid，文章和案例才使用卡片。
- 页面板块应是全宽 band 或无框布局。
- 表单和文章可以使用白底与 hairline 边框。

## 7. 圆角、边框与阴影

| Token | 值 | 用途 |
| --- | --- | --- |
| Radius 4 | 4px | 标签、输入框 |
| Radius 8 | 0px | 网格单元、按钮 |
| Radius 16 | 0px | 大型内容容器 |

规则：

- 网格单元默认无圆角。
- CTA 按钮使用 Fugu 式胶囊圆角，但不要把标签、卡片、内容容器都做成胶囊。
- 边框优先于重阴影。
- 阴影非常克制，只用于悬浮层、菜单；常规信息模块不使用阴影。

阴影：

```css
--shadow-soft: 0 8px 24px rgba(15, 23, 42, 0.06);
--shadow-pop: 0 12px 32px rgba(15, 23, 42, 0.12);
```

## 8. 组件系统

### 8.1 Header

用途：

- 全站顶部导航。

结构：

- 左侧：白泽明理 Baize Tech 字标。
- 中间或右侧：导航项。
- 右侧：预约咨询按钮。
- 移动端：汉堡菜单。

状态：

- 默认白底或透明，滚动后使用半透明白底与底部 hairline。
- 滚动后可加浅色背景和底部分隔线。
- 当前页使用 Fugu Red 或底线强调。

### 8.2 Button

类型：

- Primary：主 CTA。
- Secondary：次级按钮。
- Ghost：文本型导航按钮。

规则：

- 主按钮使用 Fugu Red 或 Ink，根据页面层级选择。
- 次按钮使用透明背景和 Ink 边框。
- 按钮高度：44px / 52px 两档。
- 图标按钮优先使用标准图标，不用文字塞满小按钮。

### 8.3 Service Card

用途：

- 展示四类服务。

内容结构：

- 服务编号。
- 服务名。
- 一句话价值。
- 适合对象。
- 交付内容摘要。
- 链接 CTA。

视觉规则：

- 白底。
- 1px Neutral 200 边框。
- Hover 时只改变边框或文字颜色，不做明显位移。

### 8.4 Method Step

用途：

- 展示诊断、接入、培训、流程、沉淀。

结构：

- 步骤序号。
- 动词标题。
- 解释文本。
- 输出物。

视觉规则：

- 可使用横向时间线或竖向流程。
- 连接线使用 Neutral 200。
- 当前强调点使用 Fugu Red、Logo Blue 或 Logo Orange，其中 Orange 优先保留给品牌圆点。

### 8.5 Article Card

用途：

- 明理笔记列表和首页精选内容。

结构：

- 分类标签。
- 标题。
- 摘要。
- 阅读时间或发布日期。

视觉规则：

- 不做复杂封面图依赖。
- 可使用低对比度纹理或符号做内容识别。
- 文章卡片标题必须清楚，不使用标题党。

### 8.6 Contact Form

字段：

- 姓名。
- 公司 / 团队。
- 联系方式。
- 当前最想解决的问题。
- 感兴趣的服务。

状态：

- 默认。
- Focus。
- Error。
- Success。

规则：

- 表单说明要简短。
- 错误提示靠近字段。
- 提交按钮文案具体，例如“提交咨询需求”。

### 8.7 Badge / Tag

用途：

- 服务类型、文章分类、状态标签。

规则：

- 不使用过多颜色。
- 默认使用 Neutral 100 背景。
- 强调标签可使用淡橙或淡蓝背景。

## 10. 内容宣传设计规则

明理笔记是品牌专业信任的核心栏目，不是普通新闻动态。

设计要求：

- 首页只放精选内容，不放完整列表。
- 内容卡片标题要像判断，不像营销。
- 文章页排版要适合长文阅读。
- 每篇文章结尾放低压 CTA，不强行打断阅读。

推荐内容卡片示例：

- 企业为什么不能只买 AI 账号？
- 小团队如何建立第一套 AI 编程工作流？
- Codex 真正进入研发流程，需要解决哪些问题？
- Skills 不是 Prompt：为什么它是组织知识资产？

## 11. 动效规则

动效要轻，服务于理解。

允许：

- Hover 轻微变色或边框变化。
- CTA 颜色过渡。
- 流程线渐进出现。
- 页面进入时轻微淡入。

避免：

- 大面积粒子动画。
- 无限循环装饰动画。
- 复杂 3D。
- 快速闪烁。

动效时长：

- 快速反馈：120ms。
- 常规过渡：180ms。
- 页面入场：320ms。

## 12. 响应式规则

断点：

- Mobile：0-767px。
- Tablet：768-1023px。
- Desktop：1024px 以上。

移动端要求：

- 导航折叠。
- Hero 标题不超过 3 行。
- 服务卡片单列。
- 表单字段单列。
- CTA 按钮宽度可满行。

## 13. 可访问性

- 文本与背景对比度满足 WCAG AA。
- 按钮和链接有清晰 focus 状态。
- 表单字段有 label。
- 不只依赖颜色表达状态。
- 图片必须有 alt。
- 动效遵守 `prefers-reduced-motion`。
