## 1. 项目背景

白泽明理是一家**设计型 AI 咨询公司**，帮助企业、小团队和个体用户理解 AI、普及 AI、落地 AI、应用 AI，不因为 AI 落后于人。

公司英文概念为 `Formal eXplainable AI`，中文品牌为「白泽明理」。「白泽」来自中国古代神话中能「通万物之情，晓天下万物状貌」的神兽，天然对应可解释 AI 的品牌隐喻：理解复杂系统，并用人能听懂的方式解释它。「明理」强调透明、可解释、数理逻辑、严谨证明和可落地的方法。

白泽的交付物是**方法与设计**，不是运维与代码：给出可验证的架构、流程、规范与诊断，让客户团队据此落地。这与「明理」品牌一致：明 = 讲清楚，理 = 方法与证明。

官网第一版的核心任务不是炫技，也不是做成大而全的 AI 产品矩阵，而是建立清晰可信的品牌认知，让目标客户快速明白：白泽明理可以帮助他们安全、系统、持续地把 AI 用起来，并把这件事变成可增长、可提效的组织能力。

## 2. 官网定位与核心理念

### 2.1 一句话定位

> 白泽明理，让每个组织都能明白 AI，用好 AI，不因为 AI 落后于人。

英文辅助定位：

> Formal eXplainable AI for practical AI adoption.

### 2.2 核心理念

> 宣传 AI 能力，推动企业在 AI 工具上应用落地，助力企业增长和效率提升。

这句话决定了官网所有内容的落点：一切服务与内容最终都要指向**增长**与**效率**两个可感知结果，而不是停在「技术很先进」。

### 2.3 三层价值表达

官网应表达三层递进价值，正好对应服务体系的三层阶梯：

1. **接入（解释清楚 + 稳定可控）**：帮助用户理解 AI 的能力、边界、风险，并建立安全可信的 AI 使用入口与网关。
2. **工程（真实落地 + 可验证）**：把 AI 编程、研发流程、迭代回路设计成可控、可验证、可复用的工程能力。
3. **交付（增长与效率）**：设计从想法到产品的端到端交付路径，让 AI 能力转化为真实产出与增长。

### 2.4 参考方向

- 可参考 Sakana AI 首页的极简品牌气质、强符号首屏、克制导航结构。
- 不照搬 Sakana AI 的研究机构定位；白泽明理应更偏企业服务、落地方法、可信解释。

## 3. 服务体系

白泽的五项主服务不是并列的清单，而是一条从「接入」到「交付」的**价值阶梯**。客户可以从最低摩擦的中转站进入，被顺势带到高价值的端到端交付设计。全部服务的交付口径统一为：**设计 + 规范 + 蓝图 + 指导**，白泽不下场代做。

```
接入层 Access        Codex 中转站搭建  →  LLM API 网关设计
                     先把 AI 稳定、可控、可审计地接进来
                            ↓
工程层 Engineering    AI 研发流设计  →  Loop Engineering 设计
                     再把 AI 变成可迭代、可验证的工程能力
                            ↓
交付层 Delivery       想法到产品 · 端到端交付设计
                     最终把能力设计成真实产品与增长路径
```

支撑层（贯穿全程，不单列为主服务卡）：**AI 编程培训（随工工作坊）** 与 **Skills 与知识沉淀（工程输出）**。

### 3.1 接入层

#### 3.1.1 Codex 中转站搭建

服务目标：

- 为企业或团队设计安全可信的 Codex 接入入口。
- 解决 Codex 访问稳定性、团队共享、权限管理、使用边界等问题。
- 降低团队分散使用外部 AI 工具带来的不可控风险。

交付物（设计口径）：接入架构方案、安全与权限规则、使用边界规范、落地与运维指导手册。

页面表达重点：

- 不说成「代理工具」或「账号服务」，不出现「账号代购」「代理」等词。
- 表达为「企业 AI 使用入口」「可信中转站」「组织级 AI 落地的基础设施设计」。
- 强调稳定、可控、可审计、可扩展。

#### 3.1.2 LLM API 网关设计

服务目标：

- 把中转站泛化为统一的多模型 API 网关设计。
- 解决统一鉴权、多模型路由与编排、限流、成本核算、可观测（日志/追踪）、故障降级、密钥托管、合规审计等平台级问题。

交付物（设计口径）：网关架构蓝图、模型路由与降级策略、成本与配额方案、可观测与审计设计、选型建议。

页面表达重点：

- 面向平台/基础架构团队与技术决策者。
- 「可审计、可控」两个品牌承诺在这一层第一次变成具体能力。
- 强调这是组织级 AI 基础设施，而不是某个工具的接入脚本。

### 3.2 工程层

#### 3.2.1 AI 研发流设计

服务目标：

- 把 AI 编程纳入真实研发流程，而不是个人自由发挥。
- 让 AI 参与需求拆解、方案设计、代码实现、测试、评审和文档，形成从需求到交付的协作规范。

交付物（设计口径）：AI 研发流程诊断、Codex / Claude Code / Cursor 使用规范、需求拆解模板、代码生成与审查流程、测试与验收流程、AI 研发 SOP 文档。

页面表达重点：

- 面向研发负责人与小团队负责人。
- 强调「流程」而不是「工具推荐」，结果是组织效率与交付质量。

#### 3.2.2 Loop Engineering 设计

服务目标：

- 设计 AI 工作的迭代回路：让 AI 系统能自我验证、自我纠错、持续改进，而不是一次性生成。
- 覆盖 eval 驱动的开发回路、Agent 的「计划-执行-观察-纠正」回路、生产反馈回路。

交付物（设计口径）：回路架构设计、评估（eval）体系与指标、反馈与纠错机制、可复用的回路模板。

页面表达重点：

- 这是白泽的差异化招牌，也是品牌承诺「可验证」压力最大的一层。
- 因为概念最难懂，必须配一句人话解释（例如：让 AI 像有经验的工程师一样，先自查再交付，每一轮都比上一轮更准）。
- Skills 与知识沉淀是这一层的自然输出：把有效回路固化成可复用资产。

### 3.3 交付层

#### 3.3.1 想法到产品端到端交付设计

服务目标：

- 用前面四层能力，设计一条从想法到可用产品的端到端交付路径。
- 定位为**纯咨询**：白泽设计交付流程、里程碑、架构与验收标准，执行由客户团队完成，白泽不下场代做产品。

交付物（设计口径）：端到端交付蓝图、阶段里程碑与验收标准、角色与协作分工、风险与边界说明。

页面表达重点：

- 这是最能证明白泽专业深度、也最贴近「增长与效率」理念的一层。
- 措辞是「交付设计」，强调白泽给的是可执行的交付路径与方法，而不是外包代码。
- 是首页价值阶梯的终点，视线与 CTA 都收束到这里。

### 3.4 支撑层（不单列为主服务卡）

- **AI 编程培训**：作为咨询与交付过程中的随工工作坊、企业内训或公开课，帮助个人、研发团队和小团队从工具使用升级到真实项目工作流。是获客与赋能的低门槛入口，也是个体用户的主要触点之一，但不作为并列主服务展示。
- **Skills 与知识沉淀**：作为研发流设计与 Loop Engineering 的可复用资产输出。把团队规范、业务经验和交付方法写成可复用的 AI Skills，是组织知识资产，不是简单 prompt。在服务页作为「输出 · 可复用资产」呈现，不单列。

## 6. 内容与文案原则

### 6.1 文案气质

文案应克制、清晰、可信，不使用过度营销词。

推荐语气：解释型、专业但不晦涩、有判断、直接指出用户真实问题、少说概念多说落地结果。

避免：颠覆行业、赋能万物、革命性效率提升、一站式 AI 解决方案、空泛的「AI 时代已经到来」。

### 6.2 技术服务的「讲人话」要求

新阵容里 LLM API 网关、Loop Engineering 等术语偏技术。因为受众四类并重且品牌承诺「明理 = 讲清楚」，每个技术服务标题下必须配一句面向非技术读者的人话解释。示例：

- LLM API 网关 = 团队所有 AI 调用的统一、安全、可计费的总入口。
- Loop Engineering = 让 AI 像有经验的工程师一样，先自查再交付，每一轮都更准。
- 端到端交付设计 = 一条从想法到可用产品、每步都能验收的路线图。

### 6.3 关键表达

推荐表达：

- 让复杂 AI 变得可解释、可验证、可落地。
- 不只是买工具，而是设计组织级 AI 使用能力。
- 把个人经验沉淀成团队可复用的 Skills。
- 帮团队从尝鲜 AI 走向稳定使用 AI。
- 安全、稳定、可控地接入 AI。

不推荐表达：账号代购、代理服务、普通培训机构、大模型供应商、AI SaaS 平台、外包代开发。

## 7. SEO 与内容引擎

第一版把明理笔记作为主要获客与 SEO 引擎，权重高于 v1。原则：

- 每一项主服务对应一个话题簇（pillar + 支撑文章），文章内链回服务页。
- 标题围绕真实搜索意图：Codex 中转、LLM 网关、AI 研发流程、agent loop 工程、AI 端到端交付。
- 基础 SEO 信息（title / description / 语义化结构 / Open Graph）在第一版即需具备。
- 个体用户以内容触达为主，是内容引擎的核心受众池。

## 8. 转化路径

主要转化目标（均为咨询/诊断，不含代开发承诺）：

- 预约 AI 落地诊断。
- 企业培训合作。
- 中转站 / 网关服务咨询。
- Loop / 端到端交付路径共创咨询。

转化入口：首页首屏、服务卡片、明理笔记文章结尾、方法论页结尾、全站页脚。

咨询表单提交后建议展示：

- 已收到需求。
- 会在 1-2 个工作日内联系。
- 可提示用户补充当前 AI 使用场景。

## 10. 验收标准

第一版官网通过以下标准即为成功：

- 用户 10 秒内能看懂白泽明理是做什么的。
- 用户能清楚看出五项服务是一条「接入 → 工程 → 交付」的阶梯，而不是零散清单。
- 用户能理解白泽是「设计型咨询，只设计不下场」，不误认为外包开发。
- 「白泽」品牌记忆点强，不只是普通 AI 工具站。
- 明理笔记能支撑专业信任并具备基础 SEO。
- 每个关键页面都有明确咨询入口。
- 移动端阅读顺畅，表单路径清楚。
- 视觉风格克制、可信、有东方神话识别度，不像普通蓝紫色 AI SaaS 模板。
